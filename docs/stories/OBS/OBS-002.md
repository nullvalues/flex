## Context

The SPA refactor of Phase G (agreements `HARNESS007-main.md` DP1/DP6): the UI is reworked to
read the OBS-001 resolver state model — per-leaf-worker waypoints, the resolver index, and
next-action state — instead of the retired orchestrator-centric signals. After this story the
old signal reads are gone from the routes.

## Requires

- OBS-001 complete: `resolver-state --json` + the `resolverState.ts` reader and the route
  source switch.

## Ensures

- The SPA UI (`App.tsx` + the relevant panels) renders from the resolver state model: the
  current next-action (action/scalar/model/reason), the Position (active phase, next story,
  gate signals, `checkpoint_step`, `needs_spec`), per-**leaf-worker** effort/waypoints (keyed
  by `agent_role`, not by an orchestrator step), and the resolver-owned index (deferred/backlog
  shown inactive, CER-056).
- The **orchestrator-centric signal reads are removed** from `system.ts` / `context.ts`
  (replaced by the OBS-001 source); no route re-derives loop state.
- Loopback-only default is preserved (CER-042 posture unchanged); no new cross-origin exposure.
- The API builds/typechecks; the UI builds. A route-level test (or the existing API test
  harness) asserts the system/context routes return the resolver-model shape and no longer
  reference the removed signals.
- `PATH=$HOME/.local/bin:$PATH uv run pytest tests/pairmode/ -x -q` passes (Python side
  unaffected; TS build/tests green).

## Instructions

- Drive every panel from `resolverState.ts`; delete the dead orchestrator-signal code paths
  in the two routes once the UI no longer references them.
- Keep per-leaf-worker waypoints aligned with the effort.db `agent_role` rollup OBS-001
  exposes — do not invent a new grouping.
- Preserve loopback-only default and the existing security posture (CER-042/043 are addressed
  in INFRA-166, not loosened here).
