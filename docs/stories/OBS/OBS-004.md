## Context

D2/CER-054 (agreements `HARNESS007-main.md` DP4; supersedes CER-045 symptom 2).
`context_current_tokens` shows the 25k SessionStart reset seed and is stale fleet-wide; the
live PostToolUse writer (`post_tool_use.py`, matcher `Task|Agent`, reading the JSONL transcript)
appears not to update these projects. Under the thin harness the spawn tool / transcript shape
may differ from what the writer expects. This story root-causes and fixes the writer (or records
"idle since the Phase 74 fix" with an honest staleness indicator).

> `hooks/post_tool_use.py` is a protected file — this story names it in `primary_files` with
> this reason (the live token writer is the defect surface). The reviewer enforces the explicit
> declaration.

## Requires

- HARNESS006 complete: the thin-harness spawn/dispatch shape is the live shape the writer must
  handle.

## Ensures

- The root cause is identified and recorded: writer not firing on the dispatch-loop spawns vs.
  projects idle. If a writer bug: `post_tool_use.py` fires on the thin-harness leaf spawns and
  updates `context_current_tokens` + `context_current_tokens_recorded_at`; if idle: recorded
  with an honest last-updated staleness indicator surfaced by the SPA (no false "live" claim).
- `context_budget.py`'s read side honors freshness (the existing `recorded_at` vs
  `session_reset_at` staleness logic) and is not regressed.
- The SPA (`stateJson.ts` / context panel) renders a last-updated **age** so a stale value is
  visibly stale, not silently authoritative.
- `tests/pairmode/test_context_token_writer.py` asserts deterministically (synthetic JSONL
  transcript + hook input): the writer extracts the token sum and writes the counter +
  `recorded_at`; a stale value is rendered as stale. No live API call; no dependence on real
  fleet state.
- Hook integrity preserved: `post_tool_use.py` stays a thin relay (no skill import, no network)
  — the security-auditor invariant holds.
- `PATH=$HOME/.local/bin:$PATH uv run pytest tests/pairmode/ -x -q` passes.
