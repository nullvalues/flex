## Context

The foundation of Phase G (agreements `HARNESS007-main.md` DP1, era open thread #4): after the
flip the orchestrator holds no loop state, so the SPA must read the **resolver state model** —
next-action state, per-leaf-worker effort, and the resolver-owned index — rather than
orchestrator-centric signals. This story adds the single pure-read source
(`flex_build.py resolver-state --json`) and a TS reader the API routes consume, so the SPA
never re-derives loop state.

## Requires

- HARNESS001–005 complete: `infer_position` / `resolve_next_action` / `ACTIONS` and the
  per-leaf effort rows (effort.db keyed by `agent_role`).

## Ensures

- **`flex_build.py resolver-state --json --project-dir .`** (additive — freeze green) emits a
  pure-read JSON document: (a) the current `make_action` dict
  (`action`/`scalar`/`model`/`reason`/`meta`/`meta.schema_version`); (b) the `infer_position`
  Position (active phase, next story, attempt count, gate signals, `checkpoint_step`,
  `needs_spec`, `last_attempt_outcome`); (c) per-leaf-worker effort summary from effort.db
  keyed by `agent_role` (builder/reviewer/gate/loop-breaker/security/intent/docs/spec); (d)
  the resolver-owned index (era/phase/story graph) with **deferred/backlog treated as inactive**
  (CER-056).
- The command is **pure-read** (`grep` confirms no writes) and composes the existing resolver
  read-model (no duplicated logic).
- `skills/observability/api/src/readers/resolverState.ts` shells the CLI (or reads its JSON)
  and returns a typed model; `system.ts` and `context.ts` read from it (the old
  orchestrator-centric signal reads are removed in OBS-002 — this story adds the new source and
  switches the two routes' primary reads).
- `test_cli_surface_freeze.py` stays green (additive command).
  `tests/pairmode/test_resolver_state_cli.py` asserts the JSON shape against synthetic
  resolver-fixture trees (next-action state present; Position fields present; per-role effort
  summary present; deferred phase reported inactive).
- `PATH=$HOME/.local/bin:$PATH uv run pytest tests/pairmode/ -x -q` passes; the API
  typechecks/builds.

## Instructions

- Build `resolver-state` as a thin CLI over `infer_position`/`resolve_next_action` + an
  effort.db per-role rollup — reuse, do not reimplement. Treat deferred/backlog as inactive
  in the index read (CER-056; this is the same rule HARNESS008 consolidates — share it if
  `index_integrity` exists, else a local helper).
- Keep the TS reader honest about staleness (surface the source timestamps) for OBS-004's D2
  work.
- Do not refactor the SPA UI here (OBS-002); only add the source + switch the two routes'
  primary reads.
