## Ensures

- **CER-044** — `phaseIndex.ts` validates that the resolved filesystem path for each
  phase file (constructed from the `href` in `index.md`) is contained within `projectDir`
  before attempting to read it. A crafted `index.md` with `[Phase](../../../../etc/passwd)`
  does not cause the API to read outside the project directory.
- The containment check uses Node.js `path.resolve()` + `.startsWith(path.resolve(projectDir))`.
- On a containment failure: the phase entry is skipped (same as a missing file); no error
  is thrown to the caller.
- `PATH=$HOME/.local/bin:$PATH uv run pytest tests/pairmode/test_observability_parsers.py -x -q`
  passes (or equivalent TypeScript test if applicable).
- `PATH=$HOME/.local/bin:$PATH uv run pytest tests/pairmode/ -x -q` passes.

## Instructions

In `skills/observability/api/src/parsers/phaseIndex.ts`, after constructing the candidate
path from `href`:

```typescript
const candidatePath = path.resolve(projectDir, 'docs', 'phases', href);
const safeRoot = path.resolve(projectDir);
if (!candidatePath.startsWith(safeRoot + path.sep) && candidatePath !== safeRoot) {
  // traversal detected — skip this entry
  continue; // or return null
}
```

Apply this check before the file read at lines 44-46.

Add a test case in `test_observability_parsers.py` (or a TypeScript test) verifying that
