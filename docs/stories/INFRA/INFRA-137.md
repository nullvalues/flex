# INFRA-137 ‚Äî `flex_build.py permissions create`

## Background

The current per-story allow-rule cycle (`write-permissions` ‚Üí builder ‚Üí `clear-permissions`)
writes to `.claude/settings.local.json`. That path is inside `.claude/`, which the Claude Code
auto-mode classifier hard-blocks unconditionally, requiring user intervention on every story.

The fix is to materialise each story's declared scope into a JSON permissions file under
`docs/phases/permissions/`. That directory is already protected from builder writes by the
existing `Edit(docs/phases/**)` deny rule, so no new `settings.json` entries are needed.
The file is generated by a `flex_build.py permissions create` Bash call ‚Äî which writes to
`docs/phases/`, not `.claude/` ‚Äî and therefore runs in auto mode without user approval.

## Ensures

### `permissions create` subcommand

- `flex_build.py permissions create` is registered as a Click command on the `flex_build`
  group, accepting:
  - `STORY_ID` positional argument (e.g. `INFRA-137`)
  - `--project-dir PATH` (default `"."`, `click.Path(file_okay=False, dir_okay=True)`)
- The RAIL is derived from the story ID prefix (`INFRA-137` ‚Üí `INFRA`;
  `BUILD-024` ‚Üí `BUILD`). Pattern: everything before the first `-NNN` suffix.
- The story spec is read from `<project_dir>/docs/stories/<RAIL>/<STORY_ID>.md`.
  If the file does not exist the command prints an error to stderr and exits non-zero.
- `primary_files` and `touches` are extracted from YAML frontmatter via the existing
  `_read_story_frontmatter` helper (already used elsewhere in `flex_build.py`).
  Missing or empty fields are treated as empty lists; no error is raised.
- The story spec's own relative path (`docs/stories/<RAIL>/<STORY_ID>.md`) is always
  appended to `allowed_paths` (so the builder can update `status: complete`).
- `allowed_paths` is the deduplicated union of `primary_files + touches + [story_spec_path]`,
  preserving declaration order with the story spec path last.
- The output file is written to
  `<project_dir>/docs/phases/permissions/<STORY_ID>.json` with the structure:
  ```json
  {
    "story_id": "INFRA-137",
    "story_spec": "docs/stories/INFRA/INFRA-137.md",
    "allowed_paths": ["..."],
    "generated_at": "<ISO-8601 UTC timestamp>"
  }
  ```
- `<project_dir>/docs/phases/permissions/` is created if it does not exist.
- The command is idempotent: re-running overwrites the file cleanly.
- On success, stdout prints one line:
  `permissions: wrote docs/phases/permissions/<STORY_ID>.json (<N> paths)`
- Exit 0 on success; exit 1 if the story spec is not found or frontmatter cannot be parsed.

### Module docstring + registration

- The `flex_build.py` module-level docstring gains `permissions create` in its command list.
- `permissions create` is registered via `@flex_build.command("permissions-create")`.
  (Dash not underscore ‚Äî matches Click's convention for multi-word flat commands.)

## Out of scope

- Removing or deprecating `write-permissions` / `clear-permissions` (BUILD-024).
- Hook enforcement of the permissions file (INFRA-138, INFRA-139).
- Glob matching in `allowed_paths` ‚Äî exact path strings only for now.
- A `permissions list` or `permissions diff` subcommand.

## Instructions

### 1. Add the command to `flex_build.py`

Add a `cmd_permissions_create` function decorated with
`@flex_build.command("permissions-create")`. Place it after `cmd_clear_permissions`
following the existing ordering pattern.

```python
import datetime as _dt  # add to module-level imports if not present

@flex_build.command("permissions-create")
@click.argument("story_id")
@click.option(
    "--project-dir",
    default=".",
    type=click.Path(file_okay=False, dir_okay=True),
    help="Project root directory.",
)
def cmd_permissions_create(story_id: str, project_dir: str) -> None:
    """Generate docs/phases/permissions/<STORY_ID>.json from story spec frontmatter."""
    project_path = Path(project_dir).resolve()
    rail = story_id.split("-")[0]
    story_spec_rel = f"docs/stories/{rail}/{story_id}.md"
    story_path = project_path / story_spec_rel

    if not story_path.exists():
        click.echo(f"permissions-create: story spec not found: {story_path}", err=True)
        sys.exit(1)

    try:
        fm = _read_story_frontmatter(story_path)
    except Exception as exc:
        click.echo(f"permissions-create: failed to parse frontmatter: {exc}", err=True)
        sys.exit(1)

    primary_files: list[str] = fm.get("primary_files") or []
    touches: list[str] = fm.get("touches") or []

    seen: set[str] = set()
    allowed: list[str] = []
    for p in primary_files + touches:
        if p not in seen:
            seen.add(p)
            allowed.append(p)
    if story_spec_rel not in seen:
        allowed.append(story_spec_rel)

    out_dir = project_path / "docs" / "phases" / "permissions"
    out_dir.mkdir(parents=True, exist_ok=True)
    out_path = out_dir / f"{story_id}.json"

    payload = {
        "story_id": story_id,
        "story_spec": story_spec_rel,
        "allowed_paths": allowed,
        "generated_at": _dt.datetime.utcnow().strftime("%Y-%m-%dT%H:%M:%SZ"),
    }
    out_path.write_text(json.dumps(payload, indent=2) + "\n", encoding="utf-8")
    click.echo(f"permissions: wrote docs/phases/permissions/{story_id}.json ({len(allowed)} paths)")
```

## Tests

File: `tests/pairmode/test_flex_build_permissions_create.py`

Use `click.testing.CliRunner` and `tmp_path` (pytest fixture).
For each test, create a minimal story spec file in the tmp_path tree under
`docs/stories/<RAIL>/<ID>.md` with YAML frontmatter.

1. `test_permissions_create_writes_json_with_correct_structure`
   ‚Äî story with `primary_files: [a.py]` and `touches: [b.py]`; assert output
   file exists at `docs/phases/permissions/INFRA-999.json`; assert JSON has
   `story_id`, `story_spec`, `allowed_paths`, `generated_at` keys.

2. `test_permissions_create_includes_story_spec_in_allowed_paths`
   ‚Äî assert `docs/stories/INFRA/INFRA-999.md` is always in `allowed_paths`.

3. `test_permissions_create_deduplicates_paths`
   ‚Äî put same path in both `primary_files` and `touches`; assert it appears
   once in `allowed_paths`.

4. `test_permissions_create_handles_missing_primary_files`
   ‚Äî frontmatter with no `primary_files` key; assert exits 0 and
   `allowed_paths` contains only the story spec path.

5. `test_permissions_create_handles_missing_touches`
   ‚Äî frontmatter with no `touches` key; assert exits 0 and `allowed_paths`
   contains only the declared `primary_files` plus the story spec path.

6. `test_permissions_create_exits_nonzero_when_story_not_found`
   ‚Äî pass a non-existent story ID; assert exit code non-zero and stderr
   contains "not found".

7. `test_permissions_create_idempotent`
   ‚Äî run twice with same inputs; assert second run overwrites cleanly and
   exits 0 with correct content.

8. `test_permissions_create_registered_on_flex_build_cli`
   ‚Äî assert `"permissions-create"` in `flex_build.commands`.

9. `test_permissions_create_stdout_reports_path_and_count`
   ‚Äî assert stdout contains the `permissions: wrote` line with the correct
   path and the path count.
