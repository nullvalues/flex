## Goal

Replace the per-story `write-permissions`/`clear-permissions` cycle ŌĆö which writes allow rules into `.claude/settings.local.json` and requires user intervention in auto mode ŌĆö with a permissions-file approach that is zero-interaction and auto-mode-safe. Each story's `primary_files` + `touches` frontmatter is materialised into a JSON permissions file under `docs/phases/permissions/`. That path is already covered by the existing `Edit(docs/phases/**)` / `Write(docs/phases/**)` deny rule, so the file is immutable to builder Edit/Write tool calls without any new settings.json entries. A new `scope_guard.py` module, delegated to from an extended `pre_tool_use.py`, reads the active story's permissions file on every Edit/Write tool call and blocks writes to undeclared paths at the hook level. The orchestrator's build loop replaces the `write-permissions` Bash call with `permissions create` and drops `clear-permissions` entirely ŌĆö both operations now write to `docs/phases/` rather than `.claude/`, so they require no user approval in auto mode. A companion fix to `sync.py` closes the regression where syncs wiped the deny list from downstream projects. Era 001's INFRA rail continues shifting deterministic enforcement out of prose and settings files into versioned, testable code.
